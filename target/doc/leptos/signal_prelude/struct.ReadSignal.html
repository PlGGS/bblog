Expand description

The getter for a reactive signal.

A signal is a piece of data that may change over time, and notifies other code when it has changed. This is the core primitive of Leptosâ€™s reactive system.

ReadSignal is also Copy and 'static, so it can very easily moved into closures or copied structs.

Core Trait Implementations

  • .get() (or calling the signal as a function) clones the current value of the signal. If you call it within an effect, it will cause that effect to subscribe to the signal, and to re-run whenever the value of the signal changes.
    • .get_untracked() clones the value of the signal without reactively tracking it.
  • .with() allows you to reactively access the signalâ€™s value without cloning by applying a callback function.
    • .with_untracked() allows you to access the signalâ€™s value without reactively tracking it.
  • .to_stream() converts the signal to an async stream of values.

Examples

let (count, set_count) = create_signal(cx, 0);

// âœ… calling the getter clones and returns the value
assert_eq!(count(), 0);

// âœ… calling the setter sets the value
set_count(1);
assert_eq!(count(), 1);

// âŒ you could call the getter within the setter
// set_count(count() + 1);

// âœ… however it's more efficient to use .update() and mutate the value in place
set_count.update(|count: &mut i32| *count += 1);
assert_eq!(count(), 2);

// âœ… you can create "derived signals" with the same Fn() -> T interface
let double_count = move || count() * 2; // signals are `Copy` so you can `move` them anywhere
set_count(0);
assert_eq!(double_count(), 0);
set_count(1);
assert_eq!(double_count(), 2);
